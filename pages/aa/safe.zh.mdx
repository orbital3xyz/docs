
# 安全账户

#### 安全账户的密钥有三个部分，分别为

* 活跃密钥：用户持有，日常签名使用
* 托管密钥：平台持有，配合用户共同管理账户
* 备份密钥（可选）：用户持有，利用备份密钥可以恢复账户

### 完全托管模式

{% hint style="info" %}
密钥组成：托管密钥&#x20;
{% endhint %}

将账户委托给平台管理，方便易用。

### 共同托管模式

{% hint style="info" %}
密钥组成：活跃密钥+托管密钥
{% endhint %}

_通过添加外部密钥升级成为共同托管模式，钱包开发商**无法擅自操作**用户的账户。_

在该模式下，用户和平台各持有一份密钥，一方面可以限制平台作恶，还可以避免由于用户密钥泄漏带来的安全问题。 通常配合第三方云存储使用，避免丢失风险。

### 辅助托管模式

{% hint style="info" %}
密钥组成：活跃密钥+托管密钥+备份密钥&#x20;
{% endhint %}

_继续添加恢复密钥到安全账户，此时，用户拥有了对于账户的完全控制权，钱包开发商**无法阻止用户操作**自己的账户。_&#x20;

这个模式也可以称为自托管模式，因为用户对账户的安全负有绝对的责任，密钥分布为平台拥有一份密钥，用户持有两份。当系统停止运营或者出现不可控因素时，用户可以利用备份密钥，取回账户的控制权。
